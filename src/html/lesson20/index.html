HTML - Урок 16

HTML- формы

Форма HTML используется для сбора вводимых пользователем данных. Пользовательский ввод чаще всего отправляется на сервер для обработки.






Элемент <form>

Элемент HTML <form> используется для создания HTML-формы для ввода данных пользователем:

<form>
.
Элементы формы
.
</form>

Элемент <form> представляет собой контейнер для различных типов элементов ввода, таких как: текстовые поля, флажки, переключатели, кнопки отправки и т. д.



Элемент <input>

Элемент HTML <input> является наиболее часто используемым элементом формы.

Элемент <input> может отображаться разными способами, в зависимости от type атрибута.

Тип Описание
<input type="text"> Отображает однострочное поле ввода текста
<input type="radio"> Отображает переключатель (для выбора одного из многих вариантов)
<input type="checkbox"> Отображает флажок (для выбора нуля или более из множества вариантов)
<input type="submit"> Отображает кнопку отправки (для отправки формы)
<input type="button"> Отображает кликабельную кнопку


Текстовые поля

Определяет <input type="text"> однострочное поле ввода для ввода текста.

<form>
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname">
</form>


Элемент <label>

Тег <label> связывает подпись элемента формы ( <input> , <textarea> , <select> ) с самим элементом



Радио-кнопки

Определяет <input type="radio"> переключатель.

Радиокнопки позволяют пользователю выбрать ОДИН из ограниченного числа вариантов.

<p>Choose your favorite Web language:</p>

<form>
  <input type="radio" id="html" name="fav_language" value="HTML">
  <label for="html">HTML</label><br>
  <input type="radio" id="css" name="fav_language" value="CSS">
  <label for="css">CSS</label><br>
  <input type="radio" id="javascript" name="fav_language" value="JavaScript">
  <label for="javascript">JavaScript</label>
</form>

Флажки

Флажки позволяют пользователю выбирать ОДИН или БОЛЬШЕ вариантов из ограниченного числа вариантов.

<form>
  <input type="checkbox" id="vehicle1" name="vehicle1" value="Bike">
  <label for="vehicle1"> I have a bike</label><br>
  <input type="checkbox" id="vehicle2" name="vehicle2" value="Car">
  <label for="vehicle2"> I have a car</label><br>
  <input type="checkbox" id="vehicle3" name="vehicle3" value="Boat">
  <label for="vehicle3"> I have a boat</label>
</form>

Кнопка «Отправить»

Определяет <input type="submit"> кнопку для отправки данных формы обработчику формы.

<form action="/action_page.php">
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname" value="John"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname" value="Doe"><br><br>
  <input type="submit" value="Submit">
</form>


Атрибут действия

Атрибут action определяет действие, которое будет выполнено при отправке формы.

<form action="/action_page.php">
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname" value="John"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname" value="Doe"><br><br>
  <input type="submit" value="Submit">
</form>

Целевой атрибут Target

Атрибут target указывает, где отображать ответ, полученный после отправки формы.

Значение Описание
_blank Ответ отображается в новом окне или вкладке
_self Ответ отображается в текущем окне
_parent Ответ отображается в родительском фрейме
_top Ответ отображается в полном тексте окна
framename Ответ отображается в именованном iframe

Пример

<form action="/action_page.php" target="_blank">

Атрибут метода

Атрибут method указывает метод HTTP, который будет использоваться при отправке данных формы.

Данные формы могут быть отправлены как переменные URL (с помощью method="get" ) или как почтовая транзакция HTTP (с помощью method="post" ).

Метод HTTP по умолчанию при отправке данных формы — GET. 

Примеры

<form action="/action_page.php" method="get">
<form action="/action_page.php" method="post">

Совет: Всегда используйте POST, если данные формы содержат конфиденциальную или личную информацию!


Атрибут автозаполнения (autocomplete)

Атрибут autocomplete указывает, должно ли в форме быть включено или выключено автозаполнение.

<form action="/action_page.php" autocomplete="on">

Атрибут Novalidate

Атрибут novalidate является логическим атрибутом.

<form action="/action_page.php" novalidate>

Если он присутствует, он указывает, что данные формы (входные данные) не должны проверяться при отправке.

<form action="/action_page.php" novalidate>

Список всех атрибутов <form>

Атрибут Описание
accept-charset Указывает кодировки символов, используемые для отправки формы.
action Указывает, куда отправлять данные формы при отправке формы.
autocomplete Указывает, должно ли в форме быть включено или выключено автозаполнение.
enctype Указывает, как данные формы должны быть закодированы при отправке на сервер (только для метода="post").
method Указывает метод HTTP, который будет использоваться при отправке данных формы.
name Указывает имя формы
novalidate Указывает, что форма не должна проверяться при отправке.
rel Указывает связь между связанным ресурсом и текущим документом.
target Указывает, где отображать ответ, полученный после отправки формы.

Домашнее задание: Для формы

  1. Создайте веб-страницу с HTML-формой для ввода данных. Ваша форма должна содержать следующие элементы:

    • Текстовое поле для ввода имени (используйте <input type="text">).
    • Текстовое поле для ввода фамилии (используйте <input type="text">).
    • Группу радиокнопок для выбора любимого языка программирования (HTML, CSS, JavaScript).

  2. Добавьте подписи к каждому из элементов формы, используя элемент <label>. Свяжите каждую метку с соответствующим элементом формы с помощью атрибутов for и id.

  3. Создайте кнопку отправки формы (используйте <input type="submit>).

  4. Укажите атрибут action в вашей форме так, чтобы данные формы были отправлены на сервер (вы можете использовать какой-либо фиктивный URL в качестве значения action).

  5. Определите атрибут method для вашей формы. Выберите один из методов: "get" или "post". Объясните, когда лучше использовать каждый метод.

  6. Используйте атрибут target в форме для определения, где будет отображаться ответ после отправки формы. Выберите один из вариантов: "_blank", "_self", "_parent", "_top", или укажите имя фрейма, если у вас есть именованный iframe.

  7. Добавьте кнопку "Сбросить" в форму (используйте <input type="reset>), чтобы пользователи могли сбросить все введенные данные.

  8. Установите атрибут novalidate в форме. Объясните, какие ситуации могут потребовать отключения встроенной браузерной проверки формы.

  9. После заполнения формы, отправьте ее (если у вас есть серверная часть, убедитесь, что она обрабатывает данные). Если у вас нет сервера, попробуйте отправить форму на какой-либо фиктивный URL или создайте обработчик формы в JavaScript.
